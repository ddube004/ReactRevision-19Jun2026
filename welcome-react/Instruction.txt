# React Notes

## 1. React Creates SPA (Single Page Application)

* React builds SPA applications.
* Only the required part of the page updates.
* Full page reload is not required.

```text
React = SPA
```

---

## 2. Project Start

Command:

```bash
npm start
```

Description:

* Starts the React development server.
* Internally runs:

```bash
react-scripts start
```

---

## 3. First File Loaded

```text
public/index.html
```

Contains:

```html
<div id="root"></div>
```

Description:

* Root container of the React application.
* React loads all components inside this div.

---

## 4. Entry Point (Root Logic File)

```text
src/index.js
```

Description:

* First JavaScript file executed by React.
* Tells React:

  * Which component to render.
  * Where to render it.

Example:

```jsx
root.render(<App />);
```

Meaning:

```text
Render App component
inside
<div id="root"></div>
```

---

## 5. React Flow

```text
npm start
    ↓
react-scripts start
    ↓
public/index.html
    ↓
<div id="root">
    ↓
src/index.js
    ↓
root.render(<App />)
    ↓
App.js
    ↓
Browser
```

---

## 6. Access DOM Elements

### By ID

```javascript
document.getElementById("id")
```

Example:

```javascript
document.getElementById("root")
```

Used to access an element using its id.

---

### By Query Selector

```javascript
document.querySelector("selector")
```

Examples:

```javascript
document.querySelector("#root")
document.querySelector(".box")
document.querySelector("h1")
```

Used to access the first matching element.

---

## 7. Component Data

Point to Remember:

```text
Every Component has its own data and logic.
```

Example:

```jsx
function Counter() {
  const [count, setCount] = useState(0);
}
```

Description:

* Data inside a component belongs to that component only.
* One component cannot directly access another component's state.

---

## 8. React Router DOM Hierarchy

Memory Trick:

```text
Grandparent → BrowserRouter
Parent      → Routes
Child       → Route
```

Structure:

```jsx
<BrowserRouter>
  <Routes>
    <Route path="/" element={<Home />} />
  </Routes>
</BrowserRouter>
```

Description:

* BrowserRouter → Enables routing.
* Routes → Holds all routes.
* Route → Maps URL to Component.
